Spark is an open source cluster computing system that aims to make data analytics fast - both fast to run and fast to write - achieved through the in-memory data load and querying.

Spark in memory concept provide boost up for iterative algorithms (machine learning algorithms) and interactive data mining making it faster (100x) than traditional Hadoop MapReduce.
Spark also provide enhanced support for Scala programming ( Integration to Scala & interactive use from Scala interpreter ).



Acyclic data flowto Resilient Distributed Datasets (RDDs)

The current cluster programming models are based on acyclic data flow from stable storage to stable storage. The benefit leveraged of following such a data flow pattern is that, the runtime can decide where to run the tasks and can automatically recover from failures.
The acyclic data flow is inefficient for applications that repeatedly reuse a working set of data like the Iterative algorithms (machine learning, graphs) & Interactivedata mining tools (R, Excel, Python).
Moreover, with the current frameworks, data reload happens from stable storage on each query execution which hinders performance.


Resilient Distributed Datasets (RDDs) 

RDD concept is built to keep the working sets in memory for efficient reuse and keeping in mind on how to retain the best features of MapReduce like Fault tolerance, data locality & scalability.
                                                                                                                                                                                                                                                        
The main highpoints on Resilient distributed datasets (RDDs) programming model are :
Immutable, partitioned collections of objects.
Created through parallel transformations (map, filter, groupBy, join, …) on data in stable storage.
Data can be cached for efficient reuse.
Actions on RDDs include- Count, reduce, collect, save, etc.

Spark provides clean, concise APIs in Scala, Javaand Python. Also Spark can be used interactively from the Scala and Python shells to rapidly query big datasets.

Spark Caching Sample

val file = spark.textFile(“hdfs://…”)
val errors = file.filter(line => line.contains(“ERROR”))

// Count all the errors  in the log file
errors.count()

// Count errors mentioning MySQL
errors.filter(line => line.contains(“MySQL”)).count()

// Fetch the MySQL errors as an array of strings
errors.filter(line => line.contains(“MySQL”)).collect()

//cache the error messages in RAM
errors.cache() 

Here after the first action that uses errors, later ones will be much faster.
