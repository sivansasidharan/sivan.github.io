Amazon Machine Image (AMI) Copy. 

AMI Copy enables us to easily copy your Amazon Machine Images between AWS Regions.

This feature can help to easily and consistently deploy instances in multiple regions, thus enabling to build better performing and more highly available applications.

AMI Copy helps enable several key scenarios including:

  1. Simple and Consistent Multi-Region Deployment - Can copy an AMI from one region to another, enabling to easily launch consistent instances based on the same AMI into different regions.
  2. Scalability - Can more easily design and build world-scale applications that meet the needs of users, regardless of their location.
  3. Performance - Can increase performance by distributing application and locating critical components of the application in closer proximity to the users and also take advantage of region-specific features such as instance types or other AWS services.
  4. Even Higher Availability - Can design and deploy applications across AWS regions, to increase availability. 
Copies can be initiated from the AWS Management Console , the command line tools, the EC2 API or the AWS SDKs.





There are no additional charges for AMI copy beyond the bandwidth and storage used for the copy and AMI at rest. A much awaited & big leap forward !
