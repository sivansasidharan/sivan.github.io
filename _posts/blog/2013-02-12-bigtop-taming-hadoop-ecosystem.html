BigTop is the outcome of the pressing demand for the management of multiple distributions for Hadoop and its related ecosystem projects. Installing and configuring Hadoop and its components can become a nightmare as version compatibility issues could pop-up like – ‘an x version of HBase might not be compatible with the installed Hadoop y version’.
Apache Bigtop brings an end to such issues by providing a fully integrated and tested Big Data management distribution.

Why BigTop ?

Bigtop is a project for the development of packaging and tests of the Apache Hadoop ecosystem. It aims at Packaging, Deployment & Integration Testing of all the sub-projects of Hadoop. This includes testing at various levels (packaging, platform, runtime, upgrade, etc...) with the primary focus on the system as a whole, rather than individual projects.
(http://bigtop.apache.org/)

Building from BigTop
Packages have been built on Ubuntu 10.10, CentOS 5, Mageia 1 and openSUSE 11.4. They can probably be built on other platforms as well. Installing Bigtop Hadoop distribution artifacts lets you have an up and running Hadoop cluster complete with various Hadoop ecosystem projects in just a few minutes.

Below points define the steps for setting it up in an Ubuntu environment (bigtop-0.4.0-incubating)–

a.       Install the Apache Bigtop GPG key – Installs the GNU Privacy Guard key, a key management system to access all public key directories.
        wget -O- http://www.apache.org/dist/bigtop/bigtop-0.4.0-incubating/repos/GPG-KEY-bigtop | sudo apt-key add –
 
b.      Grab and update the sources list ( the repository files )
  sudo wget -O /etc/apt/sources.list.d/bigtop.list http://www.apache.org/dist/bigtop/bigtop-0.4.0-incubating/repos/ubuntu/bigtop.list
 
c.  Enable the mirror that is closest to you (uncomment one and only one pair)
        sudo vi /etc/apt/sources.list.d/bigtop.list
 
d.      Update the apt cache
     sudo apt-get update
 
e.      Browse through the artifacts
                    apt-cache search hadoop
 
f.        Make sure the latest version of Oracle JDK is installed in the system.
 
g.       Install the full Hadoop stack (or parts of it)
     sudo apt-get install hadoop\* flume-* mahout\* oozie\* whirr-*
 
Running Hadoop 

                    After the necessary packages are installed – 
 
a.       Format the namenode – 
sudo -u hdfs hadoop namenode –format
 
b.      Start the necessary Hadoop services (in pseudo-distributed mode as below) - 
for i in hadoop-namenode hadoop-datanode hadoop-jobtracker hadoop-tasktracker ; do sudo service $i start ; done
 
 
Running Hadoop Components 

Bigtop provides the ease of installation of the different Hadoop Components without having to hunt for a specific Hadoop Component distribution and matching it with a specific Hadoop version.
 
For example:
o    Pig Install - 
sudo apt-get install pig
 
o    Hbase Install - 
sudo apt-get install hbase\*
sudo service hbase-master start
